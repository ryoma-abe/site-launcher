# インポート / エクスポートガイド

Site Launcher に保存されているサイト一覧は、設定ページから JSON 形式でバックアップ（エクスポート）および復元（インポート）できます。このガイドでは手順と注意点を紹介します。

## エクスポート手順

1. Chrome のツールバーから Site Launcher を開き、右上の歯車アイコンをクリックして設定ページを表示します。
2. 画面右上の「エクスポート」ボタンを押します。
3. `site-launcher-backup-YYYY-MM-DDTHH-MM-SS.json` というファイルがダウンロードされます。安全な場所に保管してください。

### エクスポートファイルの構造

```json
{
  "version": 1,
  "exportedAt": "2025-11-03T12:34:56.789Z",
  "sites": [
    { "name": "Google", "url": "https://google.com", "key": "G" }
  ]
}
```

- `version`: フォーマットのバージョン番号です。
- `exportedAt`: エクスポートを行った日時（ISO8601）。
- `sites`: 保存されているサイト一覧です。

## インポート手順

1. 設定ページ右上の「インポート」ボタンを押し、バックアップした JSON ファイルを選択します。
2. ファイルの解析と検証が行われ、問題なければ既存の登録内容が置き換わります。
3. 正常に読み込まれると「サイトのインポートが完了しました」というメッセージが表示されます。

### インポート時の注意

- JSON の形式が正しくない場合や、サイトの情報が空の場合はエラーになります（保存内容は変更されません）。
- ショートカットキーは重複しないように自動で正規化されます。重複があった場合は先に出現したエントリが優先されます。
- 既存データは読み込んだファイルで上書きされます。マージしたい場合は、エクスポート → JSON 編集 → インポート の手順でファイルを調整してください。

## トラブルシューティング

| 症状 | 対応策 |
| --- | --- |
| エラー「JSON の解析に失敗しました」 | ファイルが壊れていないか確認し、テキストエディターで内容を確認してください。 |
| エラー「インポートできるサイトがありませんでした」 | `sites` 配列が空か、要素の形式が無効です。名前・URL・キーが揃っているか確認してください。 |
| インポート後にショートカットが足りない | 36個（A-Z,0-9）を超えるサイトは登録できません。不要なサイトを削除してから再度実行してください。 |

## ベストプラクティス

- 大きな変更を行う前に必ずエクスポートしてバックアップを残してください。
- バックアップファイルはバージョン管理システムやクラウドストレージに保存しておくと復旧が容易です。
- 異なる端末間で共有する際は、共有先で **Chrome に同じアカウントでログイン** していることを確認してください。
